Ejemplo 5: Datos tabulares y estructura semántica

¿Cuándo Usar Tablas?

✅ Uso Correcto

  • Datos financieros y reportes
  • Horarios y calendarios
  • Comparaciones de productos
  • Estadísticas y métricas
  • Cualquier información que se organice naturalmente en filas y columnas

❌ Uso Incorrecto

  • Layout de página (usar CSS Grid/Flexbox)
  • Alineación de elementos (usar CSS)
  • Diseño responsive (complicado con tablas)

Beneficios de la Estructura Semántica

  • thead: Agrupa encabezados de columna
  • tbody: Contiene el cuerpo principal de datos
  • tfoot: Agrupa información de resumen (totales, promedios)
  • caption: Proporciona título descriptivo

Atributos de Accesibilidad

  • scope="col": Indica que es encabezado de columna
  • scope="row": Indica que es encabezado de fila
  • caption: Proporciona contexto sobre el contenido
  • Los lectores de pantalla usan esta información para navegar

Mejores Prácticas para Tablas

Estructura

  • Usar caption para describir el contenido
  • Agrupar con thead, tbody, tfoot
  • Usar th para encabezados, td para datos

Accesibilidad

  • Incluir atributo scope en encabezados
  • Usar caption o summary para contexto
  • Mantener estructura simple cuando sea posible

Responsive

  • Envolver en contenedor con overflow-x: auto
  • Considerar alternativas para móviles (listas, cards)
  • Usar CSS para ocultar columnas menos importantes

Nota: Las tablas son para datos tabulares, no para layout. Para diseño de página, usa CSS Grid o Flexbox.

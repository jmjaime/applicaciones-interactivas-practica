Controles de Selección

Radio Buttons (Una Opción)
Checkboxes (Múltiples Opciones)
<!-- Radio buttons (una opción) --> <input type="radio" id="pequena" name="tamano" value="pequena"> <label for="pequena">Pizza Pequeña</label> <!-- Checkboxes (múltiples opciones) --> <input type="checkbox" id="queso" name="extras" value="queso"> <label for="queso">Queso Extra</label>

Listas Desplegables

<select name="pais"> <option value="">Selecciona un país</option> <option value="ar">Argentina</option> <option value="br">Brasil</option> </select> <!-- Select múltiple --> <select name="idiomas" multiple> <option value="es">Español</option> <option value="en">Inglés</option> </select>

Validación HTML5

Formulario con Validación
<!-- Validación automática --> <input type="email" required> <input type="password" minlength="8" maxlength="20"> <input type="number" min="18" max="99"> <input type="text" pattern="[A-Z]{3}[0-9]{3}">

Nota: Los campos cambian de color según su validez (rojo = inválido, verde = válido)

Mejores Prácticas

Accesibilidad

  • Labels asociados: Usar for e id
  • Fieldset y legend: Agrupar campos relacionados
  • Mensajes claros: Usar title y placeholder
  • Orden lógico: Navegación con Tab coherente

UX y Usabilidad

  • Campos requeridos: Marcar con asterisco (*)
  • Validación en tiempo real: Feedback inmediato
  • Placeholder útil: Ejemplos, no instrucciones
  • Agrupación lógica: Usar fieldset para organizar

Seguridad

  • HTTPS: Siempre para datos sensibles
  • Validación servidor: No confiar solo en HTML5
  • Sanitización: Limpiar datos de entrada
  • Tokens CSRF: Proteger contra ataques

Nota: Los formularios son la principal forma de interacción en aplicaciones web. Siempre prioriza la accesibilidad y usabilidad.
